DATABASE MIGRATION INSTRUCTIONS
================================

You need to add new columns to the upcoming_events table.

METHOD 1: Using Batch File (Recommended)
-----------------------------------------
1. Open Command Prompt
2. Navigate to backend folder:
   cd backend

3. Run the batch file:
   update-upcoming-events-table.bat


METHOD 2: Using MySQL Command Line
-----------------------------------
1. Open Command Prompt
2. Login to MySQL:
   mysql -u root -p

3. Select your database:
   USE your_database_name;

4. Run these commands one by one:

   ALTER TABLE upcoming_events ADD COLUMN location VARCHAR(255) DEFAULT NULL AFTER description;
   ALTER TABLE upcoming_events ADD COLUMN start_date DATE DEFAULT NULL AFTER location;
   ALTER TABLE upcoming_events ADD COLUMN end_date DATE DEFAULT NULL AFTER start_date;
   ALTER TABLE upcoming_events ADD COLUMN link_url TEXT DEFAULT NULL AFTER end_date;

5. Verify columns were added:
   DESCRIBE upcoming_events;


METHOD 3: Using phpMyAdmin or MySQL Workbench
----------------------------------------------
1. Open phpMyAdmin or MySQL Workbench
2. Select your database
3. Go to SQL tab
4. Copy and paste the SQL from: backend/config/update-upcoming-events-simple.sql
5. Execute


VERIFY MIGRATION
----------------
After running migration, check if columns exist:

DESCRIBE upcoming_events;

You should see these columns:
- id
- title
- description
- location          <- NEW
- start_date        <- NEW
- end_date          <- NEW
- link_url          <- NEW
- image_url
- display_order
- is_active
- created_at
- updated_at


TROUBLESHOOTING
---------------
If you get "Column already exists" error:
- This is OK! It means the column was already added.
- Just continue with the next column.

If migration fails:
- Check your .env file has correct database credentials
- Make sure MySQL server is running
- Try METHOD 2 (manual commands) instead
